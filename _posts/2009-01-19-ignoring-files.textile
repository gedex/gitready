Kita tidak membutuhkan Git untuk mem-versi-kan semua hal diproyek kita, baik itu sumber terkompilasi, berkas dengan kata sandi, atau berkas temporer yang dibuat editor. Biasanya menjaga sesuatu dari jangkauan VCS merupakan kerjaan tidak mudah dan menyebalkan untuk <em>setup</em>nya. Tapi tidak dengan Git! Menggunakan berkas @.gitignore@ dengan opsi lainnya, kita akan belajar bagaimana men-<em>setup</em> @.gitignore@ per-proyek dan perpengguna. 

p=. "!http://farm1.static.flickr.com/4/7240060_1332e880ff.jpg!":http://flickr.com/photos/miskan/7240060/

Cara tergampang dan paling sederhana adalah membuat berkas @.gitignore@ di root direktori proyek Anda. Berkas terabaikan yang didefinisikan didalamnya berlaku untuk semua direktori di dalam proyek Anda, kecuali ada @.gitignore@ lagi di dalamnya. Anda memiliki satu tempat untuk mengkonfigurasi pengabaian, tidak seperti svn:ignore pada SVN yang harus di set di setiap folder. Dan juga, berkas @.gitignore@ dapat diversikan, yang tentunya hal bagus.

Berikut contoh dasar @.gitignore@:

<pre>
$ cat .gitignore

# Dapat mengabaikan berkas tertentu
.DS_Store

# Mentargetkan dengan wildcards juga
*~
*.swp

# Dapat mengabaikan semua direktori dan semua berkas di dalam direktori.
tmp/**/* 
</pre>

Tentunya, hal ini dapat menjadi lebih kompleks. Anda juga dapat menambahkan pengecualian dengan awalan @!@. Lihat contohnya di "Panduan Github untuk pengabaian":http://github.com/guides/ignore-for-git.

Dua hal yang perlu diingat dengan pengabaian berkas: Pertama, jika berkas sudah teramati oleh Git, menambahkan berkas tersebut ke @.gitignore@ tidak akan menghentikan Git untuk mengamatinya. Anda perlu melakukan @git rm --cached <file>@ untuk membuatnya tetap berada di-<em>tree</em> dan terabaikan. Kedua, direktori kosong tidak akan diamati oleh Git. Jika Anda ingin direktori tersebut diamati, perlu dimasukkan sesuatu didalamnya. Biasanya dengan @touch .gitignore@ (membuat file @.gitignore@) cukup membuat folder teramati oleh Git.

Anda juga dapat membuka @$GIT_DIR/info/exclude@ (@GIT_DIR@ biasanya berada di folder @.git@) dan menyunting berkas tersebut untuk pengabaian per-proyek saja. Masalahnya, perubahan tersebut tidak terekam, jadi gunakanlah jika Anda memiliki berkas pribadi yang tidak perlu dibagi dengan developer lainnya di proyek yang sama.

Opsi terkahir pengabaian folder adalah dengan menambahkan <em>ignore</em> per pengguna dengan men-<em>setting</em> opsi @core.excludefiles@ di berkas konfigurasi. Anda dapat men-setup @.gitignore@ di direktori HOME yang berpengaruh terhadap semua repositori Anda dengan menjalankan perintah berikut:

@git config --global core.excludesfile ~/.gitignore@

Baca "manpage":http://www.kernel.org/pub/software/scm/git/docs/gitignore.html jika Anda ingin belajar lebih dalam tentang bagaimana @git-ignore@ bekerja. Seperti biasa, jika Anda memiliki tip terkait @git-ignore@ beritahu kami melalui komentar.
