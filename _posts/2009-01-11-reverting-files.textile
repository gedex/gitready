Topik ini merupakan sumber kebingungan bagi pemakai git dikarenakan adanya beberapa cara untuk untuk megembalikan file ke kondisi awal. Mari kita tengok beberapa perintah dasar yang mungkin dibutuhkan untuk meng-<em>undo</em> perkerjaan Anda. 

Jika ingin mengembalikan satu file ke kondisi awal:

@git checkout <file>@

Masalah timbul jika Anda memiliki nama file yang sama dengan nama <em>branch</em> yang ada. Karena perintah <code>checkout</code> digunakan  untuk mengembalikan files dan juga untuk berpindah <em>branch</em>, makanya Anda perlu sintaks berikut (terima kasih, "Norbauer":http://norbauer.com/notebooks/code/notes/git-revert-reset-a-single-file):

@git checkout -- <file>@

Jika Anda ingin mengabaikan atau membuang semua perubahan yang telah dilakukan, ada dua cara untuk melakukannya.

@git checkout -f@ atau @git reset --HARD@

Dengan perintah ini maka Anda akan kehilangan semua pekerjaan yang belum di-<em>commit</em> ke direktori Anda, jadi berhati-hatilah menggunakan perintah tersebut.

Perlu diperhatikan bahwa, "'git revert' tidak sama dengan 'svn revert'!":http://bryan-murdock.blogspot.com/2007/07/git-revert-is-not-equivalent-to-svn.html @git-revert@ digunakan untuk membalik commit, akan di bahas ditip lain berikutnya.
